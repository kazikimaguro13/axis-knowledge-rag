# RAG 運用のコスト試算

RAG のランニングコストは大きく 2 ブロックに分かれる。インデックス側 (埋め込み生成) と、推論側 (検索ヒットを差し込んだ LLM 呼び出し)。前者は基本的に「初回 + 差分更新」のスポット課金、後者は「リクエスト数 × 入出力トークン」の継続課金で、性質が異なる。

埋め込み側の試算は素直だ。Gemini text-embedding-004 を例にすると、1M トークンあたり数十円〜程度に収まる。1 ドキュメント 2,000 字 (≒ 1,500 トークン) を 10,000 件入れても、初回コストは数百円で済む。差分更新は変更ファイルだけを再埋め込みするので、運用フェーズではほぼ無視できる規模になる。

推論側はモデル選択と Top K で大きく振れる。Claude Sonnet クラスで、入力 5,000 トークン (システム + 検索結果 5 件) + 出力 500 トークンとすると、1 リクエストあたり数円〜十数円のオーダー。月 1,000 リクエストで数千円〜2 万円程度を見積もっておくと安全だ。

本プロジェクトは local-first を掲げているため、開発時は DUMMY モードで API 課金ゼロにできる。production で課金が走るのは、API キーを設定して実際にユーザーが叩いたときだけだ。
