# YAML frontmatter によるメタデータ設計

Markdown の冒頭に `---` で囲んだ YAML ブロックを置く形式は、Jekyll や Hugo など静的サイトジェネレータで広まったメタデータ表現だ。プレーンテキストの可搬性を保ちつつ、機械可読な構造を持たせられるのが利点で、本プロジェクトのナレッジ表現にも採用している。

設計のコツは「軸 (axes) と自由タグ (tags) を分けること」。軸は事前に定義したスキーマで、enum や型を持つ。一方の tags は語彙統制なしで自由に付けられる。検索 UI ではこの 2 つを別系統で扱い、軸はファセットフィルタ、tags は補助的なキーワード検索に回す。

もう一つ重要なのが、`id` と `refs` による相互参照。`id` は不変の一意キーで、ファイルをリネームしても破壊されない。`refs: [doc_001]` のような形で他ドキュメントを参照しておけば、後段で参照グラフを構築したり、壊れリンク検出ができる。本プロジェクトでは Week 2 で参照整合性チェック (`integrity.py`) を導入する予定。
