# axis-knowledge-rag の将来ロードマップ

このドキュメント自体が「ToDo 軸」のサンプル兼、リポジトリの将来計画を兼ねる。ナレッジベースに ToDo を入れておくと、検索画面から将来計画にいつでもアクセスできて便利だ、という設計上の主張も含んでいる。

v0.1.0 (2026-05-18) は MVP。Streamlit UI で軸検索 + RAG 検索が動くところまで。Docker 一発起動とサンプル 10 本拡充で「30 秒で価値が伝わる」状態にする。doc_008 で論じた「自前実装」方針が崩れていないか、依存追加のたびに見直す。

v0.2.0 (2026-05-25) は差別化機能。日本語表記ゆれ吸収 (NFKC + カナ統一)、参照整合性チェック (broken refs / orphans / cycles 検出)、マーカー方式の長文ハイライト。Week 2 で「ただの検索」から「ナレッジ運用ツール」に脱皮する。

v0.3.0 (2026-06-01) は UI/UX 全面刷新で、Next.js + FastAPI 構成に移行。Streamlit はプロトタイプ専用と割り切り、SPA としての操作性 (キーボードショートカット・URL 共有) を獲得する。doc_002 で言及したベクトル検索周りもサーバ側に分離される。

v0.4 以降は Embedder / LLM プラグイン化、マルチユーザー、クラウドデプロイ手順整備。コアの「軸 × ベクトル」設計は据え置いたまま、用途別に拡張する想定。
